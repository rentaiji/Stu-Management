# 学生综合服务与教学协同平台 - 项目文档

## 一、项目概述

### 1.1 项目名称
学生综合服务与教学协同平台（STU_Manage）

### 1.2 项目简介
一个基于 Spring Boot + Vue.js 的综合性教育管理平台，支持教务管理、教师工作台、学生服务三大核心模块。系统采用 RBAC 权限模型，实现多角色（管理员、教务老师、授课老师、学生）的差异化功能访问。

### 1.3 技术栈

**后端技术：**
- Java 17
- Spring Boot 2.7.18
- MyBatis-Plus 3.5.3.1
- MySQL 8.0.33
- Druid 1.2.20（数据库连接池）
- JWT 0.9.1（身份认证）
- Hutool 5.8.22（工具类库）
- Lombok 1.18.30

**前端技术：**
- Vue.js 2.7.14
- Element UI 2.15.14
- Axios 1.6.2
- 纯静态 HTML 页面（无构建工具）

**开发环境：**
- JDK 17+
- Maven 3.6+
- MySQL 8.0+
- IDE：IntelliJ IDEA / VS Code

## 三、核心功能模块

### 3.1 用户管理与权限控制

#### 3.1.1 角色体系
| 角色 | userType | 说明 | 主要功能 |
|------|----------|------|----------|
| 学生 | '0' | 普通学生 | 选课、查看成绩、个人信息管理 |
| 教师 | '1' | 授课老师 | 课程管理、成绩录入、科研申报 |
| 管理员 | '2' | 系统管理员 | 用户管理、课程管理、权限分配 |
| 教务老师 | '1' + 权限 | 有 `academic:course:release` 权限的教师 | 课程发布、科研审核、成绩审核 |

**注意：** 教务老师本质上是教师（userType='1'），通过权限标识 `academic:course:release` 区分。

#### 3.1.2 权限模型
- **RBAC（Role-Based Access Control）**：基于角色的访问控制
- **权限标识示例：**
  - `academic:course:release` - 课程发布权限（教务老师）
  - `academic:score:input` - 成绩录入权限
  - `student:list` - 学生名单查看权限
  - `research:view` - 科研管理权限

#### 3.1.3 核心表
- `sys_user` - 用户表（统一存储所有角色）
- `sys_role` - 角色表
- `sys_menu` - 菜单权限表
- `sys_user_role` - 用户角色关联
- `sys_role_menu` - 角色菜单关联

### 3.2 课程管理模块

#### 3.2.1 课程库管理（管理员）
- 创建课程（课程代码、名称、学分、学时）
- 编辑/删除课程
- **级联删除：** 删除课程时自动清理排课记录和学生选课记录

#### 3.2.2 课程发布（教务老师）
**功能流程：**
1. 从课程库选择课程
2. 为每门课程分配授课教师
3. 设置容量、上课时间、地点
4. 发布到当前学期

**去重规则：** 同一课程在同一学期只能发布一次

**核心表：**
- `edu_course` - 课程基础信息
- `edu_course_release` - 课程发布记录
- `edu_semester` - 学期信息

#### 3.2.3 学生选课
- 查看已发布的可选课程
- 选课/退课（容量限制）
- 查看已选课程列表

**核心表：**
- `edu_student_course` - 学生选课记录

### 3.3 成绩管理模块

#### 3.3.1 成绩录入（授课老师）
- 按课程查看学生名单
- 录入平时成绩（30%）、期中成绩（20%）、期末成绩（50%）
- 自动计算总评成绩
- 批量保存/单个保存

#### 3.3.2 成绩审核（教务老师）
- 查看待审核成绩列表
- 通过/打回（需填写原因）
- 查看历史审核记录

**核心表：**
- `edu_score` - 成绩表
- `edu_grade` - 年级信息

### 3.4 科研申报与审核模块

#### 3.4.1 科研申报（教师）
支持 5 种科研类型：

**1. SCI论文**
- 论文题目、发表期刊、作者信息、通讯作者
- SCI分区、CCF分区
- 是否推荐期刊、卓越领军期刊
- 青大是否第一单位、是否高被引、是否ESI论文
- 奖励级别、论文归属、积分

**2. 卓越期刊论文**
- 论文题目、发表期刊、作者信息、通讯作者
- 期刊类型（领军/卓越）
- 青大是否第一单位、论文归属、积分

**3. CCF会议论文**
- 论文题目、会议名称、一作、通信作者
- 会议类型（CCF A/B/C）
- 青大是否第一单位、论文归属、积分

**4. 科研奖励**
- 奖励名称、成果名称、获奖级别、获奖等级
- 获奖日期、发证机关、证书号
- 所有完成人、单位排名、主要成果

**5. 科研项目**
- 项目编号、项目名称、项目来源、负责人
- 签署年度、合同签订/截止日期
- 合同金额、到账金额

**共同字段：** 备注（所有类型）

#### 3.4.2 科研审核（教务老师）
- 查看所有待审核申请
- 查看详情（结构化展示，非JSON）
- 通过/拒绝（拒绝需填写原因）
- 查看审核历史

**核心表：**
- `edu_research_application` - 科研申请主表
- `edu_research_sci_paper` - SCI论文详情
- `edu_research_excellent_paper` - 卓越期刊论文详情
- `edu_research_ccf_paper` - CCF会议论文详情
- `edu_research_award` - 科研奖励详情
- `edu_research_project` - 科研项目详情

**设计特点：**
- 主表+详情表模式，避免单表字段过多
- 外键级联删除，确保数据一致性
- 根据 `research_type` 字段区分类型

### 3.5 学生信息管理

#### 3.5.1 学生档案
**基本信息（sys_user表）：**
- 学号、姓名、性别、出生日期
- 身份证号、民族、政治面貌
- 手机号、邮箱、生源地、家庭地址

**学籍信息（edu_student_ext表）：**
- 院系、专业、班级、年级
- 入学年份、学制、学历层次
- 入学日期、预计毕业日期
- 学籍状态

**扩展信息（edu_student表）：**
- 籍贯、出生地
- 学习方式（全日制/非全日制）
- 紧急联系人、联系电话

#### 3.5.2 学生管理功能
- 新增/编辑/删除学生
- 搜索查询（学号/姓名）
- 批量导入（预留接口）

### 3.6 竞赛申报模块

#### 3.6.1 功能说明
- 学生提交竞赛申报
- 教师/管理员审核
- 查看申报历史

**核心表：**
- `edu_competition` - 竞赛申报表

### 3.7 部门与组织架构

#### 3.7.1 部门管理
- 学校、学院、系三级结构
- 部门增删改查
- 部门类型标识（0-学校，1-学院，2-系）

**核心表：**
- `sys_department` - 部门表

## 五、API 接口规范

### 5.1 统一返回格式
```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

**状态码：**
- 200 - 成功
- 401 - 未登录/Token过期
- 403 - 无权限
- 500 - 服务器错误

### 5.2 核心接口列表

#### 用户认证
- `POST /sys/user/login` - 登录
- `GET /sys/user/permissions/{userId}` - 获取用户权限

#### 课程管理
- `GET /edu/course/list` - 查询课程列表
- `POST /edu/course` - 新增课程
- `PUT /edu/course` - 更新课程
- `DELETE /edu/course/{courseId}` - 删除课程（级联）

#### 课程发布
- `GET /edu/course-release/list` - 查询已发布课程
- `POST /edu/course-release` - 发布课程
- `PUT /edu/course-release` - 更新发布信息
- `DELETE /edu/course-release/{id}` - 关闭发布

#### 成绩管理
- `GET /edu/score/list` - 查询成绩列表
- `POST /edu/score/batch` - 批量保存成绩
- `POST /edu/score/submit` - 提交审核
- `GET /edu/score/pending-audit` - 待审核列表（教务）
- `POST /edu/score/audit` - 审核成绩

#### 科研申报
- `POST /edu/research/submit` - 提交申报
- `GET /edu/research/my-applications` - 我的申请列表
- `GET /edu/research/detail/{applicationId}` - 申请详情
- `GET /edu/research/pending-audit` - 待审核列表（教务）
- `POST /edu/research/audit` - 审核申请

#### 学生管理
- `GET /sys/user/list` - 用户列表
- `POST /sys/user` - 新增用户
- `PUT /sys/user` - 更新用户
- `DELETE /sys/user/{userId}` - 删除用户

## 八、开发规范

### 8.1 后端规范

#### 命名规范
- 实体类：`EduXxx`（教育相关）、`SysXxx`（系统相关）
- Controller：`XxxController`
- Service：`XxxService` + `XxxServiceImpl`
- Mapper：`XxxMapper`

#### 返回格式
统一使用 `Result<T>` 封装：
```java
return Result.success(data);     // 成功
return Result.error("错误信息");  // 失败
```

#### 权限校验
```java
// 方式1：检查 userType
String userType = (String) request.getAttribute("userType");
if (!"2".equals(userType)) {
    return Result.error("无权操作");
}

// 方式2：检查权限标识（推荐）
Map<String, Object> permissions = userService.getUserPermissions(userId);
List<String> perms = (List<String>) permissions.get("perms");
if (!perms.contains("academic:course:release")) {
    return Result.error("无权访问");
}
```

## 十一、常见问题FAQ

### Q1: 如何添加新的权限？
**A:** 
1. 在 `sys_menu` 表插入新菜单
2. 在 `sys_role_menu` 表关联角色
3. 前端使用 `v-if="hasPermission('权限标识')"` 控制显示

### Q2: 教务老师和普通教师的区别？
**A:** 
- 本质上都是教师（userType='1'）
- 教务老师拥有 `academic:course:release` 权限
- 该权限决定能否发布课程、审核科研、审核成绩

### Q3: 为什么删除课程后还有空记录？
**A:** 
- 已修复：删除课程时会先清理学生选课记录和排课记录
- 如果仍有问题，检查数据库外键约束

### Q4: 科研申报详情显示JSON怎么办？
**A:** 
- 已修复：前端根据字段特征智能识别类型并结构化展示
- 清除浏览器缓存后重试

### Q5: 如何重置管理员密码？
**A:** 
```sql
UPDATE sys_user SET password = 'admin123' WHERE user_name = 'admin';
```
