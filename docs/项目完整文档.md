## 一、项目概述

### 1.1 项目简介

学生综合服务与教学协同平台是一个基于 Spring Boot + Vue.js 的综合性教育管理平台，整合教务管理、教师工作台、学生服务三大核心模块。系统采用 RBAC 权限模型，实现多角色差异化功能访问，支持课程管理、成绩录入、科研申报、竞赛审批、人事档案等完整业务流程。

### 1.2 技术栈

**后端技术：**
- Java 17
- Spring Boot 2.7.18
- MyBatis-Plus 3.5.3.1
- MySQL 8.0.33
- Druid 1.2.20（数据库连接池）
- JWT 0.9.1（身份认证）
- Hutool 5.8.22（工具类库）
- EasyExcel 3.3.2（Excel处理）
- Lombok 1.18.30

**前端技术：**
- Vue.js 2.7.14
- Element UI 2.15.14
- Axios 1.6.2
- 纯静态 HTML 页面（无构建工具）

**开发环境：**
- JDK 17+
- Maven 3.6+
- MySQL 8.0+
- IDE：IntelliJ IDEA / VS Code

## 三、核心功能模块

### 3.1 用户管理与权限控制

#### 3.1.1 角色体系
| 角色 | userType | 说明 | 主要功能 |
|------|----------|------|----------|
| 学生 | '0' | 普通学生 | 选课、查看成绩、个人信息管理 |
| 教师 | '1' | 授课老师 | 课程管理、成绩录入、科研申报 |
| 管理员 | '2' | 系统管理员 | 用户管理、课程管理、权限分配 |
| 教务老师 | '1' + 权限 | 有 `academic:course:release` 权限的教师 | 课程发布、科研审核、成绩审核 |

**注意：** 教务老师本质上是教师（userType='1'），通过权限标识 `academic:course:release` 区分。

#### 3.1.2 权限模型
- **RBAC（Role-Based Access Control）**：基于角色的访问控制
- **权限标识示例：**
  - `academic:course:release` - 课程发布权限（教务老师）
  - `academic:score:input` - 成绩录入权限
  - `student:list` - 学生名单查看权限
  - `research:view` - 科研管理权限

#### 3.1.3 核心表
- `sys_user` - 用户表（统一存储所有角色）
- `sys_role` - 角色表
- `sys_menu` - 菜单权限表
- `sys_user_role` - 用户角色关联
- `sys_role_menu` - 角色菜单关联

### 3.2 课程管理模块

#### 3.2.1 课程库管理（管理员）
- 创建课程（课程代码、名称、学分、学时）
- 编辑/删除课程
- **级联删除：** 删除课程时自动清理排课记录和学生选课记录

#### 3.2.2 课程发布（教务老师）
**功能流程：**
1. 从课程库选择课程
2. 为每门课程分配授课教师
3. 设置容量、上课时间、地点
4. 发布到当前学期

**去重规则：** 同一课程在同一学期只能发布一次

**核心表：**
- `edu_course` - 课程基础信息
- `edu_course_release` - 课程发布记录
- `edu_semester` - 学期信息

#### 3.2.3 学生选课
- 查看已发布的可选课程
- 选课/退课（容量限制）
- 查看已选课程列表

**核心表：**
- `edu_student_course` - 学生选课记录

### 3.3 成绩管理模块

#### 3.3.1 成绩录入（授课老师）
- 按课程查看学生名单
- 录入平时成绩（30%）、期中成绩（20%）、期末成绩（50%）
- 自动计算总评成绩
- 批量保存/单个保存

#### 3.3.2 成绩审核（教务老师）
- 查看待审核成绩列表
- 通过/打回（需填写原因）
- 查看历史审核记录

**核心表：**
- `edu_score` - 成绩表
- `edu_grade` - 年级信息

### 3.4 科研申报与审核模块

#### 3.4.1 科研申报（教师）

**支持的科研类型：**

**1. 科研论文（res_paper表）**
- **基本信息：** 论文标题、期刊名称、作者信息、通讯作者
- **分类信息：** 论文类型、会议类型
- **分区指标：** SCI分区、CCF分区
- **期刊标识：** 是否推荐期刊、期刊类型、是否卓越期刊
- **影响力：** 是否高被引、是否ESI论文、是否青大第一单位
- **奖励信息：** 奖励级别、归属、分值
- **附件：** 论文PDF、收录证明
- **其他：** 发表日期、备注

**2. 科研项目（res_project表）**
- **基本信息：** 项目编号、项目名称、项目来源
- **时间信息：** 签订年份、签订日期、结束日期
- **经费信息：** 合同金额、到账金额（万元）
- **附件：** 立项通知书、合同文件、结题证明
- **其他：** 备注

**3. 科研奖励（res_science_award表）**
- **基本信息：** 奖励名称、成果名称、奖励级别、获奖日期
- **完成人：** 全部完成人、获奖排名、单位排名
- **颁发机构：** 颁发机构、证书编号
- **主要成果：** 成果描述
- **附件：** 获奖证书、公示文件
- **其他：** 备注

#### 3.4.2 科研审核（教务老师）
- 查看所有待审核申请
- 查看详情（结构化展示，非JSON）
- 通过/拒绝（拒绝需填写原因）
- 查看审核历史

**核心表：**
- `res_paper` - 科研论文表
- `res_project` - 科研项目表
- `res_science_award` - 科研奖励表

**设计特点：**
- 独立表设计，每种类型单独建表
- 统一的审核状态字段（待审核/已通过/已驳回）
- 支持附件上传（PDF、图片等）

### 3.5 学生信息管理

#### 3.5.1 学生档案
**基本信息（sys_user表）：**
- 学号、姓名、性别、出生日期
- 身份证号、民族、政治面貌
- 手机号、邮箱、生源地、家庭地址

**学籍信息（edu_student_ext表）：**
- 院系、专业、班级、年级
- 入学年份、学制、学历层次
- 入学日期、预计毕业日期
- 学籍状态

**扩展信息（edu_student表）：**
- 籍贯、出生地
- 学习方式（全日制/非全日制）
- 紧急联系人、联系电话

#### 3.5.2 学生管理功能
- 新增/编辑/删除学生
- 搜索查询（学号/姓名）
- 批量导入（预留接口）

### 3.6 竞赛申报模块

#### 3.6.1 功能说明
- 学生提交竞赛申报
- 教师/管理员审核
- 查看申报历史

**核心表：**
- `edu_competition` - 竞赛申报表

### 3.7 人事档案管理

#### 3.7.1 档案结构

**主表（hr_employee_archive）：**
- **基本信息：** 姓名拼音、民族、年龄、出生日期、籍贯、国籍、婚否、政治面貌
- **工作信息：** 参加工作日期、从教日期、入校时间、预计退休日期、部门、当前状态
- **学历信息：** 最高学位、毕业院校、专业、学习时间、第一学历
- **职称岗位：** 专业技术职务、评定日期、职务级别、岗位、聘任等级
- **兼职信息：** 兼聘岗位、是否双肩挑、是否产业人员
- **党政职务：** 职务名称、级别、任职时间
- **教师资格：** 教师资格证号、获得日期、辅导员类别、双师型教师
- **人才称号：** 人才称号、评聘时间、引进层次、特聘层次
- **联系方式：** 手机号、邮箱、住址、紧急联系人

**子表（hr_employee_education）：**
- 学历/学位类型、毕业院校、专业、开始/结束时间
- 学位证书编号、毕业证书编号、证书路径

#### 3.7.2 管理功能
- 新建档案（关联已有教师用户）
- 编辑档案信息
- 查看档案详情
- 搜索查询（按姓名、工号）

**核心表：**
- `hr_employee_archive` - 教职工档案主表
- `hr_employee_education` - 学历学位子表

#### 自动填充
- `create_time` - 创建时间（INSERT时自动填充）
- `update_time` - 更新时间（INSERT/UPDATE时自动填充）

#### 外键约束
- 科研详情表通过 `application_id` 关联主表
- 设置 `ON DELETE CASCADE` 级联删除

## 五、API 接口规范

### 5.1 统一返回格式
```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

**状态码：**
- 200 - 成功
- 401 - 未登录/Token过期
- 403 - 无权限
- 500 - 服务器错误

### 5.2 核心接口列表

#### 用户认证
- `POST /sys/user/login` - 登录
- `GET /sys/user/permissions/{userId}` - 获取用户权限

#### 课程管理
- `GET /edu/course/list` - 查询课程列表
- `POST /edu/course` - 新增课程
- `PUT /edu/course` - 更新课程
- `DELETE /edu/course/{courseId}` - 删除课程（级联）

#### 课程发布
- `GET /edu/course-release/list` - 查询已发布课程
- `POST /edu/course-release` - 发布课程
- `PUT /edu/course-release` - 更新发布信息
- `DELETE /edu/course-release/{id}` - 关闭发布

#### 成绩管理
- `GET /edu/score/list` - 查询成绩列表
- `POST /edu/score/batch` - 批量保存成绩
- `POST /edu/score/submit` - 提交审核
- `GET /edu/score/pending-audit` - 待审核列表（教务）
- `POST /edu/score/audit` - 审核成绩

#### 科研申报
- `POST /edu/research/submit` - 提交申报
- `GET /edu/research/my-applications` - 我的申请列表
- `GET /edu/research/detail/{applicationId}` - 申请详情
- `GET /edu/research/pending-audit` - 待审核列表（教务）
- `POST /edu/research/audit` - 审核申请

#### 学生管理
- `GET /sys/user/list` - 用户列表
- `POST /sys/user` - 新增用户
- `PUT /sys/user` - 更新用户
- `DELETE /sys/user/{userId}` - 删除用户

- **解决：**

  - SQL文件使用 UTF-8 编码保存

- **解决：** 修改 `application.yml` 中的 `server.port` 配置

```java
return Result.success(data);     // 成功
```

```java
// 方式1：检查 userType
String userType = (String) request.getAttribute("userType");
if (!"2".equals(userType)) {

// 方式2：检查权限标识（推荐）
Map<String, Object> permissions = userService.getUserPermissions(userId);
List<String> perms = (List<String>) permissions.get("perms");
if (!perms.contains("academic:course:release")) {
    return Result.error("无权访问");
}
```

## 九、项目特色与创新

### 9.1 技术亮点

1. **权限复用架构**
   - 教务老师与普通教师共用 `teacher.html`
   - 通过权限标识动态渲染菜单和功能
   - 避免页面冗余，降低维护成本

2. **级联删除机制**
   - 删除课程时自动清理关联数据
   - 保证数据一致性，避免孤儿记录

3. **独立表设计**
   - 科研申报采用独立表结构（论文、项目、奖励）
   - 每种类型单独建表，字段清晰
   - 统一的审核状态管理

4. **结构化详情展示**
   - 前端根据数据类型智能渲染
   - 避免原始JSON显示，提升用户体验

5. **完整的人事档案**
   - 主表+子表结构，支持复杂档案信息
   - 涵盖基本信息、学历、职称、兼职等全方位数据

### 9.2 业务创新

1. **一体化平台**
   - 整合教务、教学、学生服务、人事档案
   - 打破信息孤岛，数据互通

2. **工作流引擎**
   - 成绩审核：教师提交 → 教务审核
   - 科研申报：教师申报 → 教务审核
   - 竞赛审批：学生申报 → 教师审批
   - 清晰的业务流程和状态流转

3. **细粒度权限控制**
   - 基于RBAC模型的权限管理
   - 支持菜单级、按钮级权限控制
   - 动态菜单渲染

## 十一、常见问题FAQ

### Q1: 如何添加新的权限？
**A:** 
1. 在 `sys_menu` 表插入新菜单
2. 在 `sys_role_menu` 表关联角色
3. 前端使用 `v-if="hasPermission('权限标识')"` 控制显示

### Q2: 教务老师和普通教师的区别？
**A:** 
- 本质上都是教师（userType='1'）
- 教务老师拥有 `academic:course:release` 权限
- 该权限决定能否发布课程、审核科研、审核成绩

### Q3: 为什么删除课程后还有空记录？
**A:** 
- 已修复：删除课程时会先清理学生选课记录和排课记录
- 如果仍有问题，检查数据库外键约束

### Q4: 科研申报详情显示JSON怎么办？
**A:** 
- 已修复：前端根据字段特征智能识别类型并结构化展示
- 清除浏览器缓存后重试

### Q5: 如何重置管理员密码？
**A:** 
```sql
UPDATE sys_user SET password = 'admin123' WHERE user_name = 'admin';
```

### Q6: 如何添加新学生？
**A:** 
1. 管理员登录后台
2. 进入“学生管理”
3. 点击“新增学生”
4. 填写基本信息和学籍信息
5. 系统会自动创建用户账号并分配学生角色

### Q7: 教师如何录入成绩？
**A:** 
1. 教师登录工作台
2. 进入“成绩管理”
3. 选择课程
4. 录入平时、期中、期末成绩
5. 系统自动计算总评
6. 保存草稿或提交审核

### Q8: 学生如何选课？
**A:** 
1. 学生登录系统
2. 进入“选课中心”
3. 浏览可选课程
4. 点击“选课”按钮
5. 系统检查容量和时间冲突
6. 选课成功后在“我的课程”中查看
