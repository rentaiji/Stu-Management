# 教师工作台功能说明

## 一、功能概述

教师工作台是教师用户的线上工作中心,集成教学、科研、管理相关工具,旨在提升工作效率。根据不同权限角色,动态显示可用功能模块。

---

## 二、核心功能模块

### 1. 首页 - 工作台概览

**功能描述:** 展示教师个人工作数据统计卡片

**显示内容:**
- 授课门数: 本学期所授课程数量
- 学生总数: 所有课程的学生总数
- 待录入成绩: 需要录入或审核的成绩数量

**权限要求:** 所有教师角色

---

### 2. 我的课程

**功能描述:** 管理所授课程的教学班信息

**主要功能:**
- 查看课程列表(课程代码、名称、学分、学时、容量、已选人数)
- 点击"查看学生"跳转到学生名单页面
- 实时显示选课人数变化

**数据源:** `edu_course_release` 表(课程发布表)

**权限标识:** `academic:course:list`

**适用角色:** 普通教师、系主任、教务管理员

---

### 3. 成绩管理

**功能描述:** 为学生录入或批量导入课程考核成绩,查看成绩分布统计

**主要功能:**
1. **单条录入**
   - 选择课程后加载学生列表
   - 分别录入平时成绩、期中成绩、期末成绩
   - 自动计算总评成绩(平时30% + 期中20% + 期末50%)
   - 保存草稿或提交审核

2. **批量录入**
   - 弹出对话框,表格形式批量输入
   - 支持Excel导入(可扩展)
   - 批量保存,减少重复操作

3. **成绩状态管理**
   - 草稿: 可修改
   - 待审核: 提交后锁定,等待教务审核
   - 已发布: 学生可见,不可修改

4. **成绩统计** (可扩展)
   - 平均分、最高分、最低分
   - 成绩分布柱状图
   - 及格率统计

**数据源:** `edu_score` 表

**权限标识:** `academic:score:input` (录入), `academic:score:audit` (审核)

**适用角色:** 
- 普通教师: 录入、提交
- 系主任/教务秘书: 审核、发布

---

### 4. 学生名单

**功能描述:** 管理所授课程的教学班学生名单,支持导出

**主要功能:**
- 按课程筛选学生
- 显示学生详细信息(学号、姓名、班级、手机号、邮箱)
- 导出CSV格式名单
- 可查看学生扩展信息(年级、专业、学籍状态等)

**数据源:** 
- `edu_student_course` (选课关系)
- `sys_user` (学生基本信息)
- `edu_student_ext` (学生扩展信息)

**权限标识:** `academic:course:list`

**适用角色:** 普通教师、系主任、辅导员

---

### 5. 科研档案

**功能描述:** 登记、维护个人的科研项目、学术论文、专利、获奖等成果,形成数字化个人学术档案

**主要功能:**
1. **成果类型**
   - 科研项目
   - 学术论文
   - 知识产权(专利)
   - 科研获奖

2. **级别分类**
   - 国家级
   - 省部级
   - 其他

3. **操作功能**
   - 新增成果(名称、类型、级别、时间、备注)
   - 编辑已有成果
   - 删除成果
   - 附件上传(可扩展)

4. **查询统计** (可扩展)
   - 按年份统计
   - 按类型筛选
   - 科研积分计算

**数据源:** `edu_teacher_research` 表

**权限标识:** `teacher:research:list`, `add`, `edit`, `delete`

**适用角色:** 普通教师、系主任

---

### 6. 个人课表 ⭐新增

**功能描述:** 查看本人授课课表,按周次显示详细安排

**主要功能:**
- 选择周次(1-18周)
- 显示课程名称、班级、上课时间、上课地点、周次范围
- 支持打印课表(可扩展)
- 课表冲突检测(可扩展)

**数据源:** `edu_course_release` 表(需包含上课时间、地点字段)

**权限标识:** `teacher:schedule:view`

**适用角色:** 普通教师、系主任

**接口示例:**
```
GET /edu/course/my-schedule?teacherId=xxx&week=5
```

---

### 7. 发布通知 ⭐新增

**功能描述:** 向负责的班级或课程学生发布通知

**主要功能:**
1. **通知对象选择**
   - 全部学生
   - 指定课程学生
   - 指定班级

2. **通知内容**
   - 标题(必填)
   - 内容(富文本,可扩展)
   - 附件(可扩展)

3. **通知管理**
   - 查看已发通知列表
   - 显示已读人数统计
   - 撤回通知(限定时间内,可扩展)

4. **消息推送** (可扩展)
   - 站内消息
   - 邮件通知
   - 微信推送

**数据源:** 新建 `edu_notice` 表(需创建)

**权限标识:** `teacher:notice:send`

**适用角色:** 系主任、辅导员

**数据库设计:**
```sql
CREATE TABLE edu_notice (
    notice_id BIGINT PRIMARY KEY AUTO_INCREMENT,
    teacher_id BIGINT NOT NULL COMMENT '发布教师ID',
    title VARCHAR(200) NOT NULL COMMENT '通知标题',
    content TEXT NOT NULL COMMENT '通知内容',
    target_type VARCHAR(20) COMMENT '通知对象类型 all/course/class',
    target_id BIGINT COMMENT '目标ID(课程ID或班级ID)',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    deleted TINYINT DEFAULT 0
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='通知表';
```

---

### 8. 财务查询 ⭐新增

**功能描述:** 查询与个人相关的课时费、项目经费等财务流水信息

**主要功能:**
1. **课时费明细**
   - 学期
   - 课程名称
   - 课时数
   - 课时费标准(元/课时)
   - 金额
   - 发放状态(已发放/待发放)
   - 总计统计

2. **项目经费**
   - 项目名称
   - 经费来源(国家自然科学基金、省教育厅等)
   - 总金额
   - 已使用金额
   - 余额
   - 经费执行率

3. **数据同步**
   - 从财务系统定时同步
   - 手动刷新按钮
   - 数据更新时间显示

**数据源:** 
- 课时费: 根据 `edu_course_release` 课时计算
- 项目经费: 新建 `edu_teacher_fund` 表或与科研表关联

**权限标识:** `teacher:finance:view`

**适用角色:** 普通教师、系主任

**数据库设计:**
```sql
CREATE TABLE edu_teacher_fund (
    fund_id BIGINT PRIMARY KEY AUTO_INCREMENT,
    teacher_id BIGINT NOT NULL COMMENT '教师ID',
    project_name VARCHAR(200) NOT NULL COMMENT '项目名称',
    fund_source VARCHAR(100) COMMENT '经费来源',
    total_amount DECIMAL(12,2) COMMENT '总金额',
    used_amount DECIMAL(12,2) DEFAULT 0 COMMENT '已使用',
    balance DECIMAL(12,2) COMMENT '余额',
    start_date DATE COMMENT '开始日期',
    end_date DATE COMMENT '结束日期',
    status CHAR(1) DEFAULT '0' COMMENT '状态 0-进行中 1-已结题',
    create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    deleted TINYINT DEFAULT 0
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='教师项目经费表';
```

## 三、权限控制机制

### 1. 前端权限控制

**实现方式:** Vue条件渲染
```html
<el-menu-item index="schedule" v-if="hasPermission('teacher:schedule:view')">
    <i class="el-icon-time"></i>
    <span>个人课表</span>
</el-menu-item>
```

**权限检查方法:**
```javascript
hasPermission(perm) {
    return this.userPerms.includes(perm);
}
```

### 2. 后端权限验证

**实现方式:** JWT拦截器 + 业务逻辑校验

**示例:**
```java
// JWT拦截器提取用户ID
Long userId = (Long) request.getAttribute("userId");

// 业务层校验权限
LambdaQueryWrapper<EduCourseRelease> wrapper = new LambdaQueryWrapper<>();
wrapper.eq(EduCourseRelease::getTeacherId, userId);
long count = courseReleaseService.count(wrapper);

if (count == 0) {
    return Result.error("无权操作");
}
```

## 四、技术实现要点

### 1. 前端技术栈
- Vue.js 2.7
- Element UI 2.15
- Axios (HTTP请求)

### 2. 后端技术栈
- Spring Boot
- MyBatis Plus
- JWT认证

### 3. 关键接口

#### 获取用户权限
```
GET /sys/user/permissions/{userId}
Response: {
    "code": 200,
    "data": {
        "roleIds": [4],
        "menuIds": [21, 25, 26, 41, 42, 43, 44, 45, 47],
        "perms": ["academic:course:list", "academic:score:input", ...],
        "menus": [...]
    }
}
```

#### 获取我的课程
```
GET /edu/score/my-courses
Headers: Authorization: Bearer {token}
```

#### 获取课程学生
```
GET /edu/course/students/{courseId}
```

#### 批量保存成绩
```
POST /edu/score/batch
Body: [{ studentId, courseId, semesterId, regularScore, finalScore, ... }]
```

## 五、数据流程图

### 成绩录入流程
```
教师登录 → 选择课程 → 加载学生列表 → 录入成绩 → 保存草稿
                                        ↓
                                   提交审核 → 系主任/教务审核 → 发布成绩 → 学生可见
```

### 通知发布流程
```
教师登录 → 进入发布通知 → 填写标题内容 → 选择通知对象
                                            ↓
                                       发送通知 → 记录日志 → 学生收到消息
```

## 七、使用指南

### 首次使用
1. 联系管理员分配教师账号和角色
2. 首次登录强制修改密码
3. 根据个人角色,系统自动显示对应功能菜单
4. 如缺少某功能,请联系管理员调整权限

### 常见问题
**Q: 为什么我看不到某个菜单?**  
A: 您的角色可能没有该功能权限,请联系管理员分配。

**Q: 成绩提交后还能修改吗?**  
A: 提交审核后无法自行修改,如需更正请向教务申请。

**Q: 如何导出学生名单?**  
A: 进入"学生名单"页面,选择课程后点击右上角"导出名单"按钮。

**Q: 财务数据多久更新一次?**  
A: 财务数据由后台系统同步,通常每月更新一次。

## 八、注意事项

1. **数据安全**: 成绩、财务等敏感数据仅限授权人员访问
2. **操作审计**: 所有关键操作均记录日志,可追溯
3. **数据备份**: 系统定期备份,防止数据丢失
4. **隐私保护**: 学生个人信息严格保密,不得外泄
5. **及时录入**: 请在规定时间内完成成绩录入,避免影响学生学业
