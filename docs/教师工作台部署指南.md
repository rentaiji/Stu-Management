# 教师工作台部署指南

## 一、部署步骤

### 1. 执行数据库脚本

按顺序执行以下SQL脚本:

```bash
# 1. 创建新数据表(通知、经费、课时费)
mysql -u root -p stu_manage < src/main/resources/sql/create_teacher_workbench_tables.sql

# 2. 添加菜单权限
mysql -u root -p stu_manage < src/main/resources/sql/add_teacher_menu_permissions.sql
```

**或者手动执行:**
1. 打开MySQL客户端
2. 选择数据库: `USE stu_manage;`
3. 依次执行两个SQL文件的内容

---

### 2. 验证数据库

检查是否成功创建表和插入数据:

```sql

SHOW TABLES LIKE 'edu_%';

-- 应该看到:
-- edu_notice
-- edu_teacher_fund
-- edu_teaching_fee

-- 检查菜单权限
SELECT * FROM sys_menu WHERE menu_id IN (45, 46, 47);

-- 检查角色权限分配
SELECT r.role_name, m.menu_name 
FROM sys_role r
JOIN sys_role_menu rm ON r.role_id = rm.role_id
JOIN sys_menu m ON rm.menu_id = m.menu_id
WHERE m.menu_id IN (45, 46, 47);
```

## 三、常见问题排查

### 问题1: 菜单不显示

**可能原因:**
- 数据库权限未正确配置
- 用户角色未分配

**解决方法:**
```sql
-- 检查用户角色
SELECT ur.user_id, r.role_id, r.role_name
FROM sys_user_role ur
JOIN sys_role r ON ur.role_id = r.role_id
WHERE ur.user_id = 2; -- T001的user_id

-- 如果没有角色,分配普通教师角色
INSERT INTO sys_user_role (user_id, role_id) VALUES (2, 4);
```

### 问题2: 权限接口报错

**可能原因:**
- Mapper未注入
- 数据库连接失败

**解决方法:**
1. 检查控制台错误日志
2. 确认Mapper接口有`@Mapper`注解
3. 检查`application.yml`数据库配置

### 问题3: 前端页面空白

**可能原因:**
- Vue.js或Element UI CDN加载失败
- JavaScript错误

**解决方法:**
1. 打开浏览器开发者工具(F12)
2. 查看Console标签是否有错误
3. 检查Network标签CDN资源是否加载成功
4. 尝试更换网络或使用本地资源

### 问题4: 数据无法加载

**可能原因:**
- 后端接口未实现
- JWT token过期
- 跨域问题

**解决方法:**
1. 检查浏览器Network标签,查看接口返回状态码
2. 确认token有效: `localStorage.getItem('token')`
3. 如接口返回404,说明后端未实现,前端会使用模拟数据

## 四、生产环境部署建议

### 1. 安全加固

```yaml
# application.yml
jwt:
  secret: your-super-secret-key-change-in-production
  expiration: 86400000 # 24小时

# 启用HTTPS
server:
  ssl:
    enabled: true
    key-store: classpath:keystore.p12
    key-store-password: your-password
```

### 2. 性能优化

- 启用Redis缓存用户权限信息
- 数据库索引优化
- 前端资源压缩和CDN加速
- 启用Gzip压缩

### 3. 监控告警

- 集成Prometheus + Grafana监控系统性能
- 配置日志收集(ELK Stack)
- 设置关键操作告警(成绩修改、大量数据导出)

### 4. 备份策略

```bash
# 每日凌晨备份数据库
0 2 * * * mysqldump -u root -p stu_manage > /backup/stu_manage_$(date +\%Y\%m\%d).sql

# 保留最近30天备份
find /backup -name "stu_manage_*.sql" -mtime +30 -delete
```
