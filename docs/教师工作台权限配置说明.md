# 教师工作台权限配置说明

## 一、权限体系概述

教师工作台采用基于角色的访问控制(RBAC),不同角色的教师可以看到不同的功能模块。

## 二、教师角色分类

### 1. 普通教师 (role_id = 4)
**权限范围:**
- ✅ 我的课程 (academic:course:list)
- ✅ 成绩管理 (academic:score:input)
- ✅ 学生名单 (academic:course:list)
- ✅ 科研档案 (teacher:research:list, add, edit, delete)
- ✅ 个人课表 (teacher:schedule:view)
- ✅ 财务查询 (teacher:finance:view)
- ❌ 发布通知 (无此权限)

**适用场景:** 一线授课教师,负责教学、成绩录入、科研记录等日常工作。

---

### 2. 系主任 (role_id = 5)
**权限范围:**
- ✅ 所有普通教师权限
- ✅ 发布通知 (teacher:notice:send) - 可向全系师生发送重要通知
- ✅ 用户管理 (system:user:list, add, edit) - 管理本系教师账户
- ✅ 完整的教学管理权限

**适用场景:** 院系负责人,需要统筹管理工作,发布官方通知。

---

### 3. 辅导员 (role_id = 6)
**权限范围:**
- ✅ 用户管理 (system:user:list, add, edit) - 管理学生信息
- ✅ 成绩查看 (academic:score:list) - 仅查看,不能修改
- ✅ 发布通知 (teacher:notice:send) - 向学生发送通知

**适用场景:** 学生工作者,负责学生管理和信息发布。

---

### 4. 学校教务管理员 (role_id = 2)
**权限范围:**
- ✅ 全校教学管理
- ✅ 成绩审核与发布
- ✅ 选课管理
- ✅ 教师发展管理

**适用场景:** 校级教务管理人员,负责全局教学管理。

---

### 5. 院系教务秘书 (role_id = 3)
**权限范围:**
- ✅ 本系用户管理
- ✅ 本系课程管理
- ✅ 成绩审核
- ✅ 选课管理

**适用场景:** 院系教务工作人员,协助系主任处理日常教务。

## 三、新增菜单权限说明

### 菜单ID分配
| 菜单ID | 菜单名称 | 权限标识 | 父菜单 | 说明 |
|--------|---------|---------|--------|------|
| 45 | 个人课表 | teacher:schedule:view | 40(教师发展) | 查看本人授课课表 |
| 46 | 发布通知 | teacher:notice:send | 40(教师发展) | 向学生/班级发布通知 |
| 47 | 财务查询 | teacher:finance:view | 40(教师发展) | 查询课时费、项目经费 |

### 权限分配规则
```sql
-- 普通教师: 课表 + 财务
INSERT INTO sys_role_menu (role_id, menu_id) VALUES (4, 45), (4, 47);

-- 系主任: 全部新功能
INSERT INTO sys_role_menu (role_id, menu_id) VALUES (5, 45), (5, 46), (5, 47);

-- 辅导员: 仅通知
INSERT INTO sys_role_menu (role_id, menu_id) VALUES (6, 46);
```

## 四、功能模块详细说明

### 1. 教学管理
- **我的课程**: 查看本学期所授课程列表、容量、已选人数
- **成绩管理**: 录入平时/期中/期末成绩,批量导入,提交审核
- **学生名单**: 查看选课学生详细信息,支持导出CSV

### 2. 科研档案
- **成果类型**: 科研项目、学术论文、知识产权、科研获奖
- **级别分类**: 国家级、省部级、其他
- **功能**: 新增、编辑、删除科研成果,形成个人学术档案

### 3. 个人工作台
- **个人课表**: 按周次查看授课安排(课程、班级、时间、地点)
- **发布通知**: 向全体学生/指定课程/指定班级发送通知
- **财务查询**: 
  - 课时费明细(学期、课程、课时、标准、金额、发放状态)
  - 项目经费(项目名称、来源、总额、已用、余额)

## 五、权限初始化步骤

### 1. 执行SQL脚本
```bash
mysql -u root -p your_database < src/main/resources/sql/add_teacher_menu_permissions.sql
```

### 2. 验证权限配置
登录系统后,使用不同角色的教师账号测试:
- 普通教师(T001): 应看到 首页、我的课程、成绩管理、学生名单、科研档案、个人课表、财务查询
- 系主任: 额外增加"发布通知"菜单
- 辅导员: 仅显示 用户管理、成绩查看、发布通知

### 3. 为现有教师分配角色
在"管理控制台 → 教师管理"中:
1. 点击教师查看详情
2. 选择对应角色(普通教师/系主任/辅导员)
3. 保存后重新登录生效

## 六、前端权限控制实现

### 1. 权限加载
```javascript
// 登录时获取用户权限
async loadUserPermissions() {
    const response = await axios.get(`/sys/user/permissions/${userId}`);
    this.userPerms = response.data.data.perms; // ['academic:course:list', ...]
}
```

### 2. 菜单动态显示
```html
<el-menu-item index="schedule" v-if="hasPermission('teacher:schedule:view')">
    <i class="el-icon-time"></i>
    <span>个人课表</span>
</el-menu-item>
```

### 3. 权限检查方法
```javascript
hasPermission(perm) {
    return this.userPerms.includes(perm);
}
```

## 七、后端接口权限验证

所有接口通过JWT拦截器验证用户身份,关键业务接口需额外校验权限:

```java
// 示例: 成绩录入权限校验
@PostMapping("/edu/score")
public Result<Boolean> save(HttpServletRequest request, @RequestBody EduScore score) {
    Long userId = (Long) request.getAttribute("userId");
    
    // 验证是否为授课教师
    LambdaQueryWrapper<EduCourseRelease> wrapper = new LambdaQueryWrapper<>();
    wrapper.eq(EduCourseRelease::getCourseId, score.getCourseId())
           .eq(EduCourseRelease::getTeacherId, userId);
    long count = courseReleaseService.count(wrapper);
    
    if (count == 0) {
        return Result.error("您不是该课程的授课教师，无权录入成绩");
    }
    
    return Result.success(scoreService.save(score));
}
```

## 八、注意事项

1. **权限变更生效**: 修改角色权限后,用户需重新登录才能生效
2. **数据隔离**: 教师只能查看和操作自己相关的课程、学生、成绩数据
3. **审计日志**: 所有关键操作(成绩修改、通知发送)需记录操作日志
4. **演示模式**: 部分功能(课表、通知、财务)如后端未实现,前端会使用模拟数据展示

## 九、扩展建议

### 未来可增加的权限
- `teacher:evaluation:view` - 查看学生评教结果
- `teacher:material:upload` - 上传教学资源
- `teacher:attendance:manage` - 考勤管理
- `teacher:thesis:guide` - 论文指导(研究生导师)
- `teacher:project:apply` - 项目申报

### 细粒度权限控制
当前为菜单级权限,后续可扩展到:
- 按钮级权限(如:某些教师只能查看不能导出)
- 数据级权限(如:只能查看本学期的成绩)
- 字段级权限(如:隐藏敏感信息)
